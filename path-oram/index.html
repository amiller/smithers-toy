Path ORAM — live simulator

A binary tree of buckets drawn the ORAM way — root at the bottom, leaves on top. Every logical access reads a full root-to-leaf path into the stash, remaps the block to a fresh random leaf, and evicts blocks back toward their leaves as deep as they will go. The server only ever sees a uniformly random path.
